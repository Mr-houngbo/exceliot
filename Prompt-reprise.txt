### 📋 Plan d'Action : Scraping Pipeline 2.0

#### 1. Consolidation des Sources (Import & Data Entry)
*   **Action** : Créer un script d'importation (`import_sources.py`) pour lire les fichiers Excel et peupler la table `sources` de Supabase.
*   **Métadonnées** : Ajouter des champs comme `country` (CI/SN), `source_type` (direct/aggregator/social) et `scraping_method` (api/html/manual).
*   **Priorisation** : Marquer les sources comme `active` ou `pending` selon leur facilité technique.

#### 2. Architecture Modulaire des Scrapers
Je propose de structurer les scrapers en trois catégories dans `apps/workers/scrapers/` :
*   **`engine_adzuna.py`** (Existant) : Utilisation d'API.
*   **`engine_generic_html.py`** : Un scraper flexible qui utilise des sélecteurs CSS (stockés en DB) pour extraire les données des sites simples (ex: Educarriere).
*   **`engine_headless.py`** : Utilisation de **Playwright** pour les sites complexes qui nécessitent du rendu JavaScript.

#### 3. Workflow d'Orchestration (Celery)
*   **Tâche `dispatch_scrapers`** : Une tâche maître qui récupère toutes les sources actives et lance une sous-tâche pour chacune.
*   **Rate Limiting** : Configurer Celery pour ne pas saturer les sites cibles et éviter les bans IP (utilisation de proxies si nécessaire).

#### 4. Raffinement du Pipeline de Nettoyage
*   **Dédoublonnage Trans-Sources** : Améliorer `deduplicator.py` pour identifier qu'une offre sur Indeed est la même que sur Educarriere.
*   **Standardisation** : Mapper tous les formats de date et de salaire disparates vers le format Supabase.
